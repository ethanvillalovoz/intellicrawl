# Contributing

IntelliCrawl values inspectable claims, bounded provider behavior, reproducible artifacts, and documentation that matches the code.

## Setup

```sh
git clone https://github.com/ethanvillalovoz/intellicrawl.git
cd intellicrawl
python -m venv .venv
source .venv/bin/activate
python -m pip install -e ".[live,media,dev]"
```

## Verification

Run the complete local gate before opening a pull request:

```sh
make check
```

When changing the generated preview:

```sh
make media
```

When changing a live provider, also run one intentional smoke query with your own keys. Include the query, report status, and sanitized warnings in the pull request. Never include credentials, cached page bodies, or private output.

## Expectations

- Keep pull requests focused and explain user-visible behavior changes.
- Preserve source IDs through analysis and export paths.
- Represent unsupported claims as unknown or partial.
- Mock external services in tests; CI must not spend API credits.
- Add regression coverage for parsing, timeout, cache, and renderer changes.
- Regenerate committed examples when report schemas or renderers change.
- Update README, usage, architecture, security, and changelog material where relevant.

## Generated Artifacts

`examples/demo-report.*` is generated by `scripts/build_demo_artifacts.py`. `docs/media/intellicrawl-evidence.png` is generated by `scripts/render_preview.py`. Do not hand-edit either output.

## Issues

Bug reports should include the command, Python version, operating system, output format, and sanitized traceback or warning. State whether the issue occurs in deterministic demo mode, live mode, or both.

Please follow the [Code of Conduct](CODE_OF_CONDUCT.md).
